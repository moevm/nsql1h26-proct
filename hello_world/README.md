# Инструмент для кластеризации данных прокторинга

## Общая идея

Это веб-приложение для преподавателей, которое помогает анализировать поведение студентов во время экзаменов. Преподаватель загружает два вида отчётов: логи системы Moodle (что студент делал на странице) и данные с камеры (результаты распознавания лиц). 

Приложение автоматически собирает из разрозненных записей цельные сессии и показывает, как эти сессии группируются в кластеры. На графике сразу видно, где находится "нормальное" поведение (большие группы похожих сессий), а какие сессии выбиваются и похожи на аномалии.

## Технологический стек

- **Серверная часть (Backend):** Node.js + Express
- **База данных:** MongoDB
- **Пользовательский интерфейс (Frontend):** React с TypeScript
- **Визуализация данных:** Chart.js
- **UI-компоненты:** Gravity UI

## Функциональность приложения

### 1. Загрузка исходных данных

Преподаватель заходит в систему и видит страницу загрузки. Можно перетащить файлы мышкой или выбрать через проводник. 

**Поддерживаемые типы файлов (CSV):**

- **Логи Moodle** - записи действий студентов в системе
- **Данные распознавания лиц** - результаты анализа видео с камеры
- **Справочник студентов** - файл с идентификаторами, именами студентов и эталонными векторами их лиц (эмбеддинги)

После загрузки система проверяет корректность структуры файлов (наличие всех необходимых колонок) и отображает прогресс обработки.

### 2. Расчёт показателей по сессиям

Для сравнения сессий между собой каждая из них преобразуется в набор числовых метрик.

**Метрики на основе логов Moodle:**

- Общее количество действий студента
- Частота активности (действий в минуту)
- Распределение типов действий (просмотр vs. активное взаимодействие)
- Частота переходов между страницами

**Метрики на основе данных с камеры:**

- Время отсутствия лица в кадре
- Время присутствия постороннего лица (несовпадение с эталоном)
- Частота смены лиц в кадре
- Средняя уверенность распознавания (similarity score с эталоном)

### 3. Кластеризация

Главная аналитическая функция системы.

**Настраиваемые параметры:**

- Выбор алгоритма кластеризации (K-means, DBSCAN и др.)
- Выбор метрик для анализа (возможность включения/отключения отдельных показателей)
- Параметры алгоритма (количество кластеров, радиус близости точек и т.д.)

После запуска система группирует все сессии по схожести поведения. Сессии, находящиеся далеко от основных скоплений, помечаются как потенциальные аномалии.

### 4. Просмотр результатов

**Структура интерфейса:**

**Левая панель** - Таблица сессий
- Столбцы: имя студента, дата, номер кластера, метка аномалии
- Возможности: поиск по имени, фильтрация по кластерам

**Центральная область** - Интерактивный график
- Визуализация сессий в виде точек на плоскости
- Цветовая кодировка по номеру кластера
- Выделение аномалий контрастным цветом
- Возможности: масштабирование, всплывающие подсказки с данными студента

**Боковая/нижняя панель** - Детальная информация
- Отображается при клике на точку или строку таблицы
- Показывает все метрики выбранной сессии
- Информация об аномалиях и отклонениях

### 5. Выгрузка отчётов

Возможность экспорта результатов анализа в формате CSV:

- Полный список всех сессий с номерами кластеров
- Список только аномальных сессий
- Список студентов, чьи сессии содержат аномалии

---

**Примечание:** Конкретная реализация функций зависит от структуры и формата исходных данных, которые будут уточнены на этапе разработки.
